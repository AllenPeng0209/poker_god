## 3. Introduction / Overview

本项目旨在设计并产品化一套“把德州扑克打好”的系统课程，覆盖基础规则、概率速记、组合与范围理解、翻前与翻后决策、下注尺度、exploit 调整、现金局与锦标赛专项、以及职业化训练闭环。  

该课程不是“知识点堆叠”，而是面向实战结果的训练系统：`学习 -> 练习 -> 测验 -> 复盘 -> 修正 -> 再训练`。目标是让用户从“凭感觉打牌”升级为“范围对范围 + EV 导向”的决策模式。

## 4. Goals

- 在 24 周内提供完整课程地图（12 模块），覆盖从新手到进阶的核心能力。
- 让用户在第 4 周前建立“概率速算 + 组合数 + 基本范围语言”三项基础能力。
- 让用户在第 10 周前具备可执行的翻前默认策略与翻牌圈基线策略。
- 让用户在第 15 周前能根据牌面结构选择合理下注尺度（含 overbet 场景）。
- 让用户在第 18 周前完成对 3 类典型对手的人群 exploit 调整。
- 建立学习成效量化体系：课程完成率、测验通过率、决策准确率、复训纠错率。
- 将课程内容与训练器/AI 教练打通，实现从知识到执行的闭环。

## 5. 课程蓝图（24 周）

| 模块 | 周期 | 核心主题 | 学习产出 |
|---|---|---|---|
| M1 | 第 1 周 | 基础规则与流程 | 熟悉牌型、行动顺序、术语、结算流程 |
| M2 | 第 2 周 | 位置与底池基础 | 理解位置价值、SPR、底池赔率、隐含赔率 |
| M3 | 第 3 周 | 概率速记与 EV | 掌握 2/4 法则与常见 outs 命中率 |
| M4 | 第 4 周 | 组合与阻断牌 | 掌握 1326 组合体系与 blocker 基础 |
| M5 | 第 5 周 | 范围语言与可视化 | 能读写 range 矩阵，理解范围优势与 nut 优势 |
| M6 | 第 6-7 周 | 翻前完整体系 | 建立开局/3bet/4bet/防守基线 |
| M7 | 第 8-10 周 | 翻牌圈核心策略 | 形成 C-bet、IP/OOP 防守、牌面分类框架 |
| M8 | 第 11-13 周 | 转牌与河牌策略 | 掌握二枪、check-back、薄价值与抓诈逻辑 |
| M9 | 第 14-15 周 | 下注尺度与线条 | 能按范围与牌面选择 33/50/75/overbet |
| M10 | 第 16-18 周 | exploit 与人群模型 | 可针对不同玩家类型进行偏离 |
| M11 | 第 19-21 周 | 现金局/MTT 专项 | 深筹码线、短码、ICM、决赛桌关键策略 |
| M12 | 第 22-24 周 | 职业化训练系统 | 建立数据库复盘、solver 验证、训练计划 |

## 6. User Stories

### US-001：课程总览与学习路径
**Description:** 作为新手或进阶玩家，我希望在课程首页看到分阶段学习路径，这样我可以知道先学什么、后学什么。

**Acceptance Criteria:**
- [ ] 课程首页按 `新手起步/数学与范围/翻前实战/翻后实战/进阶盈利/职业训练营` 6 区展示。
- [ ] 每个模块卡片包含：目标、预计时长、难度、先修条件。
- [ ] 支持按推荐顺序学习与按章节跳转学习两种模式。
- [ ] 在桌面与移动端可正常展示。
- [ ] Typecheck/lint passes.
- [ ] Verify in browser using dev-browser skill.

### US-002：单课详情页与学习目标
**Description:** 作为学习者，我希望每节课都有明确学习目标和完成标准，这样我知道“学会了什么”。

**Acceptance Criteria:**
- [ ] 每节课展示：知识点、示例手牌、常见误区、课后练习。
- [ ] 每节课有“通过标准”（例如测验 >= 80 分）。
- [ ] 每节课标记预计时长与推荐复习时间。
- [ ] Typecheck/lint passes.
- [ ] Verify in browser using dev-browser skill.

**Acceptance Criteria:**

## 8. Non-Goals（Out of Scope）

- 不提供真实资金牌局托管或对战撮合。
- 不在本期覆盖 PLO 或其他扑克变体的完整课程。
- 不在本期实现真人教练市场与交易系统。
- 不在本期做线下集训管理和企业 LMS 全量能力。
- 不将 AI 建议视为唯一标准答案，保留多线策略解释空间。

## 9. Design Considerations

- 信息架构建议固定为 6 大分区：
  - `新手起步`（M1-M3）
  - `数学与范围`（M4-M5）
  - `翻前实战`（M6）
  - `翻后实战`（M7-M9）
  - `进阶盈利`（M10-M11）
  - `职业训练营`（M12）
- 每节课页面建议包含四区块：`核心结论`、`策略解释`、`示例手牌`、`课后练习`。
- 训练和测验页面需突出计时、正确率、EV 影响，弱化无关视觉噪音。
- 必须在移动端保证可读性和关键交互可达性（课程学习与测验答题优先）。

## 10. Technical Considerations

- 建议数据模型：
  - `course_tracks`、`course_modules`、`course_lessons`
  - `quiz_questions`、`quiz_attempts`
  - `drill_sessions`、`mistake_notebook`
  - `graduation_assessments`、`learning_reports`
- 范围矩阵建议采用标准 13x13 索引，支持权重序列化存储。
- 概率训练需统一公式引擎（2/4 法则、赔率阈值）以避免前后不一致。
- 训练与测验结果需可追溯到课程版本，避免内容更新导致历史数据失真。
- AI 教练输出需附带上下文来源（模块 ID、题目 ID、范围版本）。
- 关键性能要求：
  - 课程页首屏可交互时间 < 2.5s（P75）。
  - 单次测验提交与评分反馈 < 1.5s（P75）。

## 11. Success Metrics

- 课程激活率：注册后 48 小时内进入首课的用户占比 >= 65%。
- M1-M3 完成率 >= 55%，M1-M6 完成率 >= 35%。
- 周测验平均通过率 >= 70%，复训后二次通过率 >= 80%。
- 概率速记训练平均正确率在 4 周内提升 >= 20%。
- 范围矩阵训练错误率在 6 周内下降 >= 25%。
- 毕业考核通过率（活跃用户样本）>= 30%。
- AI 教练建议被采纳并转训练任务的比例 >= 40%。

## 12. 里程碑与交付节奏

- Milestone 1（第 1-4 周）：M1-M4 + 概率训练 MVP + 基础测验。
- Milestone 2（第 5-10 周）：M5-M7 + 范围矩阵训练 + 模块晋级机制。
- Milestone 3（第 11-18 周）：M8-M10 + 错题本 + AI 教练任务化。
- Milestone 4（第 19-24 周）：M11-M12 + 毕业考核 + 成效看板。
